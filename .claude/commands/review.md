# コードレビュー <比較対象ブランチ名> <作業計画書のパス>

現在のブランチを比較対象ブランチと比較してコードレビューを行ってください。

## 引数

`$ARGUMENTS` は以下の形式で渡されます：

- 形式: `<比較対象ブランチ名> <作業計画書のパス>`
- 比較対象ブランチ名: 省略時は `main`
- 作業計画書のパス: 省略可。指定時は作業計画書との整合性もレビュー対象に含める

例:

- `/review main docs/plans/1_doing/feature.md` → main と比較、作業計画書も参照
- `/review feature-branch` → feature-branch と比較
- `/review` → main と比較

## 手順

1. **引数の解析**: `$ARGUMENTS` から比較対象ブランチ名と作業計画書のパスを取得（ブランチ名省略時は main、作業計画書は省略可）
   - `$ARGUMENTS` の最初のスペース区切りトークンを比較対象ブランチ名として使用する
   - `git branch -a` で指定されたブランチが存在するか確認する。存在しない場合はエラーを報告して終了する
   - 解析結果を明示する: 「比較対象ブランチ: `{ブランチ名}`、現在のブランチ: `{現在のブランチ名}`」
2. **差分の取得**: `git diff <比較対象ブランチ>...HEAD --name-only` で変更ファイル一覧を取得
3. **作業計画書の読み込み**（指定時のみ）: 作業計画書を読み込み、実装すべき仕様・要件を把握する
4. **レビュードキュメントの作成**: `docs/reviews/template.md` をコピーして `docs/reviews/{現在のブランチ名}-{3桁連番}.md` を作成
   - 連番は `docs/reviews/` と `docs/reviews/done/` 配下の全ディレクトリから同一ブランチ名のファイルを検索し、既存の最大連番 + 1 を付与する
   - 例: `sign-up-1-1-001.md`（同一ブランチの2回目のレビューなら `sign-up-1-1-002.md`）
5. **レビュー情報の記入**: ブランチ名、変更ファイル数、変更行数を記入。作業計画書が指定されている場合はそのパスも記入する
6. **変更ファイル一覧の記入**: 実装ファイル、テストファイル、設定ファイルに分類して記入
7. **ファイルごとのレビュー**: 各ファイルに対して以下を実行
   - ファイルを読み込む
   - テンプレートの「参照するガイドライン」セクションに記載されているガイドラインを確認
   - ファイルの種類に応じた該当ガイドラインに従っているかチェック
   - 作業計画書が指定されている場合、設計との整合性もチェック（仕様通りに実装されているか、漏れがないか）
   - 問題点があれば「ファイルごとのレビュー結果」に記録（ガイドラインへの参照を含める）
   - **問題がないファイルは「変更ファイル一覧」のチェックボックスにチェックを入れるだけでOK**（「ファイルごとのレビュー結果」に記載しない）
8. **設計との整合性チェック**（作業計画書指定時のみ）: 作業計画書に記載された要件・仕様がすべて実装されているか確認し、未実装や乖離があれば問題点として記録する
9. **総合評価の記入**: Approve / Request Changes / Comment を判定し、総評を記入
10. **完了報告**: レビュードキュメントのパスと、主要な問題点のサマリーを報告

## レビューの観点

各ファイルをレビューする際は、以下の優先順位で注意してください：

1. **設計との整合性**（作業計画書指定時）: 作業計画書の仕様・要件通りに実装されているか、漏れや乖離がないか
2. **セキュリティ**: CSRF対策、XSS対策、SQLインジェクション対策が適切か
3. **アーキテクチャ**: 依存関係のルールに従っているか
4. **既存コードとの一貫性**: 類似の処理が既存コードにある場合、そのパターンに従っているか
5. **命名規則**: ファイル名、関数名、変数名がガイドラインに従っているか
6. **エラーハンドリング**: エラーが適切に処理されているか
7. **テスト**: 実装に対応するテストが追加されているか

## 出力形式

レビュー完了後、以下の形式で報告してください：

```
## レビュー完了

**ドキュメント**: docs/reviews/{ファイル名}.md
**作業計画書**: {パス}（指定時のみ表示）

**評価**: [Approve / Request Changes / Comment]

**サマリー**:
- 問題点の数: N件
- 必須対応: N件
- 推奨対応: N件
- 設計との乖離: N件（作業計画書指定時のみ表示）

**主要な問題点**:
1. [問題点1の概要]
2. [問題点2の概要]
...
```
