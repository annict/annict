# ガイドラインの同期

## 対象プロジェクト

| プロジェクト | ディレクトリ |
|------------|------------|
| Annict     | `/workspace/` (自プロジェクト) |
| Mewst      | `/mewst/` |
| Wikino     | `/wikino/` |

※ groobb は CLAUDE.md が未整備のため同期対象外

## 比較対象ファイル

### CLAUDE.md

| 種類 | パス |
|------|------|
| 全体 | `CLAUDE.md` |
| Go版 | `go/CLAUDE.md` |
| Rails版 | `rails/CLAUDE.md` |

### Go版ガイドライン（go/docs/）

- `architecture-guide.md`
- `handler-guide.md`
- `i18n-guide.md`
- `security-guide.md`
- `templ-guide.md`
- `validation-guide.md`

## 手順

### ステップ 1: 設計書の作成

1. `docs/designs/template.md` をコピーして `docs/designs/1_doing/sync-guideline-YYYYMMDD.md` を作成する（YYYYMMDD は実行日）
2. 設計書のタイトルを「ガイドライン同期 YYYY/MM/DD」とする

### ステップ 2: 差分調査

**重要**: コンテキスト制限を回避するため、Task tool（subagent_type: `general-purpose`）を使って並列で差分調査を行うこと。

1. 比較対象ファイルについて、全プロジェクト間の差分を調査する
2. 存在しないファイルはスキップする
3. 差分がないファイルは「差分なし」と明記する

### ステップ 3: 設計書に差分レポートを記入

設計書の「設計」セクション内に、以下の形式で差分レポートを記入する：

```markdown
### 差分 1: [差分の概要]

**対象ファイル**: `go/docs/validation-guide.md`

**差分の内容**:
- **Annict**: [Annict での記述内容の要約]
- **Mewst**: [Mewst での記述内容の要約]
- **Wikino**: [Wikino での記述内容の要約]

**採用方針**:
- [ ] Annict の方式を採用（他プロジェクトに反映）
- [ ] Mewst の方式を採用（他プロジェクトに反映）
- [ ] Wikino の方式を採用（他プロジェクトに反映）
- [ ] スキップ（対応不要）
- [ ] その他: <!-- 記入 -->

---
```

### ステップ 4: タスクリストを空で用意

設計書のタスクリストセクションに、以下を記載する：

```markdown
## タスクリスト

<!-- ユーザーが採用方針を記入した後、プロジェクトごとにフェーズを分けてタスクを追加します -->

### フェーズ 1: Annict (自プロジェクト)

（採用方針の決定後に記入）

### フェーズ 2: Mewst

（採用方針の決定後に記入）

### フェーズ 3: Wikino

（採用方針の決定後に記入）
```

### ステップ 5: ユーザーへの報告

設計書の作成が完了したら、以下の内容をユーザーに報告する：

1. 作成した設計書のパス
2. 発見した差分の件数
3. 「各差分の **採用方針** のチェックボックスにチェックを入れてください。記入後にタスクリストを更新します。」というメッセージ

## 注意事項

- 存在しないファイルはスキップする
- 差分がないファイルは「差分なし」と明記する
- 自プロジェクトに該当しない内容は差分として報告しない
- 差分調査は Task tool で並列処理してコンテキスト制限を回避する
