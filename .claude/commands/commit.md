# コミットコマンド

未コミットの差分を確認し、適切な粒度でコミットを作成するためのコマンドです。

## 手順

1. `git status` で未コミットのファイルを確認
2. `git diff` で変更内容を確認（ステージングされていない変更）
3. `git diff --cached` でステージング済みの変更を確認
4. コミットを作成

- コミットの粒度: 1コミット = 1つの論理的な変更
- 変更が多岐にわたる場合は機能ごとにコミットを作成してください

### 良い粒度の例

- 「ユーザー認証のバリデーションを追加」
- 「商品一覧のN+1クエリを修正」
- 「READMEにセットアップ手順を追記」

### 粒度が大きすぎる例

- 「機能追加とバグ修正といくつかのリファクタリング」

### 分割の基準

以下の場合は別コミットに分ける：

- **Go版とRails版の変更**: 別プロジェクトの変更は分離する
- **リファクタリングと機能追加**: コード整理と新機能は分ける
- **翻訳ファイルの変更**: 可能な限り独立したコミットにする
- **アイコンやアセットの追加**: UIリソースの追加は分離する

**原則: 迷ったら細かく分割する**

### 細かすぎる分割の例（避ける）

- 同一ファイル内の関連する変更を複数コミットに分ける
- 1行の修正ごとにコミットする

## 注意事項

- コミットメッセージは[コミットメッセージのガイドライン](/CLAUDE.md#コミットメッセージのガイドライン)に従って日本語で記述
- 複数の機能やバグ修正が含まれる場合は、適切な粒度に分割してコミット
- **全ての未コミットファイルをコミットする**: ユーザーに確認せず、適切な粒度で全ての変更をコミットしてください（untracked ファイルも含む）
