# [機能名] 仕様書

<!--
**ファイルの配置ルール**:
- システム仕様書は機能名のディレクトリに `overview.md` として配置する
  - 例: `docs/system/feature-flag/overview.md`、`docs/system/datetime/overview.md`
- ユーザーが直接体験しない、システム内部の仕組み（ミドルウェア、フォーマッタ、日時表示など）を記述する
- ユーザーが直接体験する機能の仕様書は `docs/specs/` 配下に配置する
- 詳細は [@docs/README.md](/workspace/docs/README.md) を参照

**仕様書の性質**:
- 仕様書は「現在のシステムの状態」を記述するドキュメントです
- 実装が完了したら、仕様書を最新の状態に更新してください
- 過去の状態はGit履歴で参照できるため、仕様書には常に現在の状態のみを記述します

**作業計画書との関係**:
- 新しい機能の場合: `docs/private/plans/` の作業計画書に概要・要件・設計を記述し、タスク完了後にこの仕様書を作成します
- 既存機能の変更の場合: `docs/private/plans/` の作業計画書に変更内容を記述し、タスク完了後にこの仕様書を更新します

**公開時の注意事項**:
- 開発用ドメイン名を記載する場合は `example.dev` を使用してください（実際のドメイン名は記載しない）
- 環境変数の値はサンプル値のみ記載し、実際の値は含めないでください
-->
