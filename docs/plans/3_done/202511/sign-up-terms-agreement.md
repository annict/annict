## 概要

<!--
ガイドライン:

新規登録フォーム（`/sign_up`）において、現在の「利用規約とプライバシーポリシーに同意する」チェックボックスを削除し、「登録することで利用規約とプライバシーポリシーに同意したものとみなします」というメッセージ表示（Sign-in-wrap方式）に変更します。

**目的**:

- ユーザー体験の向上：チェックボックスを入力する手間を省略
- 登録フローの簡素化：フォーム入力項目を1つ削減

**背景**:

- 改正民法下では、チェックボックスの設置は必須ではない
- 「この取引には利用規約が適用される」旨を明確に表示し、利用規約へのリンクを掲載していれば法的に有効
- Google、Twitter など多くの大手サービスがこの Sign-in-wrap 方式を採用している

<!--

- チェックボックス入力欄を削除する
- 送信ボタンの近くに「登録することで、利用規約とプライバシーポリシーに同意したものとみなされます」というメッセージを表示する
- 利用規約とプライバシーポリシーへのリンクは引き続き表示する（ただし新しい配置に移動）
- チェックボックスに関するバリデーションを削除する
- 不要になった翻訳キーを削除する

### 変更対象ファイル

1. **テンプレート**: `go/internal/templates/pages/sign_up/new.templ`
   - チェックボックスの削除
   - 同意メッセージの追加（送信ボタンの直前）
   - 利用規約・プライバシーポリシーリンクの再配置

2. **リクエストDTO**: `go/internal/handler/sign_up/request.go`
   - `TermsAgreed` フィールドの削除
   - 利用規約同意バリデーションの削除

3. **ハンドラー**: `go/internal/handler/sign_up/create.go`
   - `terms_agreed` フォーム値の読み取り削除

4. **翻訳ファイル**:
   - `go/internal/i18n/locales/ja.toml`: 不要な翻訳キーの削除、新規キーの追加
   - `go/internal/i18n/locales/en.toml`: 不要な翻訳キーの削除、新規キーの追加

5. **テスト**: 既存テストファイルの更新

### UI設計

変更前:

```
[メールアドレス入力]
[✓] 利用規約とプライバシーポリシーに同意する  ← 削除
[Turnstile]
[送信ボタン]
──────────
利用規約 | プライバシーポリシー
すでにアカウントをお持ちの方は ログイン
```

変更後:

```
[メールアドレス入力]
[Turnstile]
[送信ボタン]
登録することで、利用規約とプライバシーポリシーに同意したものとみなされます。  ← 新規追加
──────────
すでにアカウントをお持ちの方は ログイン
```

### 翻訳キーの変更

**削除する翻訳キー**:

- `sign_up_agree_to_terms`: 「利用規約とプライバシーポリシーに同意する」
- `sign_up_terms_not_agreed`: 「利用規約とプライバシーポリシーに同意してください」

**追加する翻訳キー**:

- `sign_up_terms_agreement_notice`: 「登録することで、<a href="/terms" ...>利用規約</a>と<a href="/privacy" ...>プライバシーポリシー</a>に同意したものとみなされます。」
