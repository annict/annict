## 概要

<!--
ガイドライン:
- この機能が「何を」実現するのかを簡潔に説明

Go 版 Annict のログ出力方法を `log/slog` パッケージに統一する。現在、`log.Printf`/`log.Println`/`log.Fatalf` と `slog.InfoContext`/`slog.ErrorContext`/`slog.WarnContext` が混在しており、一貫性がない状態になっている。

**目的**:

- ログ出力方法を統一し、コードベースの一貫性を向上させる
- 構造化ログにより、ログの検索・分析を容易にする
- ログレベル（Info, Warn, Error, Debug）による適切な分類を実現する

**背景**:

- Go 1.21 で `log/slog` パッケージが標準ライブラリに追加され、構造化ログのデファクトスタンダードになった
- CLAUDE.md にも「`log/slog`: Go 1.21+標準構造化ログ（推奨）」と明記されている
- 既に多くの箇所で `slog` が使用されているが、一部で `log` パッケージが残っている

- すべての `log.Printf`、`log.Println`、`log.Fatalf` を適切な `slog` 関数に置き換える
- ログレベルを適切に設定する（Info, Warn, Error）
- 構造化ログ形式（キーバリュー）でログを出力する
- コンテキストが利用可能な場合は `slog.InfoContext` 等を使用する

- **運用性**: JSON 形式での出力が可能になり、ログ解析ツールとの連携が容易になる

### 変換ルール

| 変換前                              | 変換後                                           | 備考               |
| ----------------------------------- | ------------------------------------------------ | ------------------ |
| `log.Printf("メッセージ: %v", val)` | `slog.Info("メッセージ", "key", val)`            | 通常の情報ログ     |
| `log.Println("メッセージ")`         | `slog.Info("メッセージ")`                        | 引数なしの情報ログ |
| `log.Fatalf("エラー: %v", err)`     | `slog.Error("エラー", "error", err); os.Exit(1)` | 致命的エラー       |
| `log.Printf("警告: %v", msg)`       | `slog.Warn("警告", "message", msg)`              | 警告ログ           |
| `log.Printf("エラー: %v", err)`     | `slog.Error("エラー", "error", err)`             | エラーログ         |

### コンテキストの扱い

- **コンテキストが利用可能な場合**: `slog.InfoContext(ctx, ...)` を使用
- **コンテキストが利用不可能な場合**（`main.go` など）: `slog.Info(...)` を使用

### ログレベルの選択基準

| レベル       | 用途                                     |
| ------------ | ---------------------------------------- |
| `slog.Debug` | デバッグ情報（通常は出力しない）         |
| `slog.Info`  | 通常の情報（サーバー起動、処理完了など） |
| `slog.Warn`  | 警告（処理は継続するが注意が必要）       |
| `slog.Error` | エラー（処理が失敗した場合）             |
