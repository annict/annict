## 概要

<!--
ガイドライン:
- この機能が「何を」実現するのかを簡潔に説明
- ユーザーにとっての価値や背景を記述

Annict リポジトリに `CONTRIBUTING.md` ファイルを作成し、外部からのコントリビューションを受け付けていない旨を明記する。

**目的**:

- 外部コントリビューターに対して、Pull Request や機能提案を受け付けていないことを明確に伝える
- バグ報告の適切な窓口（Discord）を案内する
- セキュリティに関する報告の窓口を案内する

**背景**:

- Annict は OSS として公開しているが、小規模なチームで特定のビジョンを持って開発しているため、外部からのコントリビューションを管理するリソースがない
- ソースコードは興味のある方への公開を目的としており、共同開発を目的としていない
- GitHub の標準的な場所（`CONTRIBUTING.md`）に記載することで、Issue 作成時や PR 作成時に自動的にリンクが表示される

### 機能要件

<!--

- リポジトリのルートに `CONTRIBUTING.md` ファイルを作成する
- 以下の内容を平易な日本語で記載する：
  - Annict は現在、外部からの Pull Request を受け付けていない
  - ソースコードは興味のある方に公開しているだけであり、共同開発を目的としていない
  - バグ報告、機能提案、質問は歓迎しており、Discord で受け付けている
  - セキュリティに関する報告は `SECURITY.md` を参照するよう案内する
  - フォークは個人利用や学習目的であればライセンスの範囲内で自由に行える
