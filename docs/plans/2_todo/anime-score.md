## 概要

<!--
ガイドライン:
- この機能が「何を」実現するのかを簡潔に説明
- ユーザーにとっての価値や背景を記述
- 2-3段落程度で簡潔に

作品やエピソードに対して「このアニメは面白そうか？」という指標となるスコアを計算・表示する機能を実装する。

スコアは `animes` テーブルの `score` カラムに保存され、作品一覧やランキングページでの並び替え、おすすめ作品の表示などに活用される。計算式はユーザーに公開し、透明性を確保する。

**目的**:

- ユーザーが面白そうなアニメを発見しやすくする
- 評価の信頼性を考慮した公平なスコアリングを実現する
- 計算式を公開することで、スコアの透明性を確保する

**背景**:

- 現在、作品の評価は `satisfaction_rate`（満足度）のみで表現されている
- 評価数が少ない作品は満足度の信頼性が低いが、そのまま表示されている
- Annict のユーザー数変動（特定時期のブームなど）の影響を受けにくい指標が必要

### スコア計算の方針

#### 使用する指標

| カラム              | 説明             | 用途             |
| ------------------- | ---------------- | ---------------- |
| `ratings_count`     | 評価数           | 信頼性の重み付け |
| `satisfaction_rate` | 満足度（0-100%） | スコアのベース   |
| `score`             | 計算されたスコア | 最終的な指標     |

#### なぜ絶対数（記録数、視聴者数）を使わないか

**スコア計算に使わない理由**:

- Annict のユーザー数は時期によって変動する（例: 2018年にメディアで紹介されブームに）
- 特定時期の作品だけ記録数が異常に高くなる可能性がある
- 絶対数を使うと、古い人気作品が過大評価される

**サイトに表示しない理由**:

- 絶対数は Annict のユーザー数変動の影響を多分に受けた数値であり、作品の質を反映していない
- 知名度のあるアニメだけ数字が大きくなり、知名度のないアニメの発見に繋がらない
- SNS のフォロワー数で人を評価するような状態に似ており、本質的な価値を示さない
- ユーザーに誤解を与える可能性がある（「記録数が多い = 良い作品」ではない）

これらの理由から、記録数・視聴者数などの絶対数はスコア計算に使用せず、サイト上にも表示しない方針とする。

#### 採用する計算式: IMDb Weighted Rating 方式

評価数が少ない場合に全体平均に近づける補正を行う：

```
score = (v ÷ (v + m)) × R + (m ÷ (v + m)) × C

v = 評価数（ratings_count）
m = 最低評価数の閾値（例: 50）
R = 満足度（satisfaction_rate）
C = 全体の平均満足度
```

**この計算式の特性**:

| 評価数 | スコアの傾向                    |
| ------ | ------------------------------- |
| v = 0  | score = C（全体平均）           |
| v < m  | 全体平均に近い                  |
| v = m  | score = (R + C) / 2             |
| v > m  | 実際の満足度に近い              |
| v >> m | score ≈ R（ほぼ満足度そのまま） |

#### 既存の満足度計算ロジック

既存の `satisfaction_rate` は以下のように計算されている（`episode.rake` より）：

```ruby
# rating_state を数値に変換
# bad=0, average=1, good/great=2
ratings = rating_states.map { |state|
  case state.to_s
  when "bad" then 0
  when "average" then 1
  when "good", "great" then 2
  end
}

# 平均を 0-100% に正規化
ratings_avg = ratings.sum.to_f / ratings.length
satisfaction_rate = (ratings_avg / 2 * 100).round(2)
```

この計算は「加重平均方式」であり、`average`（普通）の評価も 50% として貢献する。

### データベース設計

`animes` テーブルに以下のカラムを追加（[animes-table.md](1_doing/animes-table.md) で定義済み）：

```sql
ratings_count integer DEFAULT 0 NOT NULL,        -- 評価数
satisfaction_rate numeric(5,2),                  -- 満足度（0.00-100.00）
score numeric(5,2),                              -- 計算されたスコア
```

### スコア計算の実装

```go
// CalculateScore は IMDb Weighted Rating 方式でスコアを計算する
func CalculateScore(ratingsCount int, satisfactionRate float64, avgSatisfactionRate float64, minRatings int) float64 {
    if ratingsCount == 0 {
        return avgSatisfactionRate
    }

    v := float64(ratingsCount)
    m := float64(minRatings)
    R := satisfactionRate
    C := avgSatisfactionRate

    return (v/(v+m))*R + (m/(v+m))*C
}
```

### バッチ処理

定期的に全作品・エピソードのスコアを再計算する：

```go
func (uc *UpdateAnimeScoresUsecase) Execute(ctx context.Context) error {
    // 1. 全体の平均満足度を計算
    avgSatisfactionRate, err := uc.animeRepo.GetAverageSatisfactionRate(ctx)
    if err != nil {
        return err
    }

    // 2. 全作品・エピソードのスコアを再計算
    return uc.animeRepo.UpdateAllScores(ctx, avgSatisfactionRate, minRatings)
}
```

### パラメータの決定

| パラメータ              | 値  | 説明                                                             |
| ----------------------- | --- | ---------------------------------------------------------------- |
| `m`（最低評価数の閾値） | 50  | この値は調整可能。大きくすると信頼性重視、小さくすると多様性重視 |

**閾値 m の選定基準**:

- 全作品の評価数の中央値や四分位数を参考に決定
- 初期値は 50 とし、運用しながら調整
