**問題点・改善提案**:

- **[@go/docs/security-guide.md#エラーメッセージ]**: エラー判定に `err.Error()` の文字列比較を使用している

  ```go
  // update.go:34
  if err.Error() == "invalid token" {
  ```

  エラーの種類判定に文字列比較を使用するのは脆弱。`fmt.Errorf("invalid token")` で生成されたエラーに対して文字列一致で判定しているが、エラーメッセージの変更に弱く、ラップされたエラーにも対応できない。

  **修正案**:

  `usecase` パッケージに sentinel error を定義し、`errors.Is` で判定する。

  ```go
  // internal/usecase/update_password_reset.go
  var ErrInvalidToken = errors.New("invalid token")

  // UseCase内でのエラー返却
  return nil, ErrInvalidToken

  // internal/handler/password/update.go
  if errors.Is(err, usecase.ErrInvalidToken) {
  ```

  **対応方針**:
  - [x] sentinel error を定義して `errors.Is` に変更する
