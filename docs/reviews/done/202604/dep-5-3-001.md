- **ロジックの忠実な移動**: Handler にあったビジネスロジック（冪等性チェック、イベントタイプ別処理、エラーハンドリング、Sentry連携）が機能的に等価な形で UseCase に移動されている
- **テスト修正**: コンストラクタ引数の変更に伴い、全テストケースのセットアップが適切に修正されている
- **main.go の変更が最小限**: UseCase の初期化を追加し、Handler のコンストラクタ引数を簡素化する最小限の変更のみ
- **ログ出力**: すべて `slog.InfoContext`/`slog.ErrorContext`/`slog.WarnContext` を使用し、コンテキスト付きの構造化ログに統一されている
