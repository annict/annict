**チェックしたガイドライン**:

- [@go/CLAUDE.md#コーディング規約](/workspace/go/CLAUDE.md) - 既存コードとの一貫性

**問題点・改善提案**:

- **既存コードとの一貫性: エラー型パターン**: `ErrAlreadyActiveSubscription` がパッケージレベルの `var ... = fmt.Errorf(...)` で定義されているが、既存の UseCase（`create_stripe_subscriber.go`, `complete_sign_up.go`）ではカスタムエラー構造体 + `Is*Error()` ヘルパー（`errors.As()` 使用）パターンが使われている。`err == ErrAlreadyActiveSubscription` による比較は、エラーが `%w` でラップされると動作しなくなるリスクがある。

  ```go
  // 現在のコード
  var ErrAlreadyActiveSubscription = fmt.Errorf("既にアクティブなサブスクリプションが存在します")

  func IsErrAlreadyActiveSubscription(err error) bool {
      return err == ErrAlreadyActiveSubscription
  }
  ```

  **修正案**:

  ```go
  // 既存パターンに合わせたカスタムエラー型
  type AlreadyActiveSubscriptionError struct{}

  func (e *AlreadyActiveSubscriptionError) Error() string {
      return "既にアクティブなサブスクリプションが存在します"
  }

  func IsAlreadyActiveSubscriptionError(err error) bool {
      var target *AlreadyActiveSubscriptionError
      return errors.As(err, &target)
  }
  ```
