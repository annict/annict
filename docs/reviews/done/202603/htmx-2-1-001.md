### レビュー内容の詳細

各 `.templ` ファイルについて以下を確認しました：

**セキュリティ（[@go/docs/security-guide.md](/workspace/go/docs/security-guide.md)）**:

- CSRF トークン（`<input type="hidden" name="csrf_token" ...>`）が全フォームに維持されている
- フォーム送信方式（`method="POST"` + `action`）に変更なく、既存の CSRF ミドルウェアがそのまま機能する
- XSS リスクなし（templ の自動エスケープ、ユーザー入力の直接表示なし）

**templ テンプレート（[@go/docs/templ-guide.md](/workspace/go/docs/templ-guide.md)）**:

- 命名規則（スネークケース）に準拠
- 国際化対応（`templates.T(ctx, ...)` の使用）が維持されている
- HTML 属性の順序がアルファベット順で一貫している

**移行パターンの一貫性**:

- 全 4 フォーム（sign_in: 1, sign_in_code: 2, sign_in_password: 1）で同一の移行パターンを適用
  - `data-on:submit__passive="$isSubmitting = true"` → `hx-on:submit="disableSubmitButtons(this)"`
  - `data-attr:disabled="$isSubmitting == true"` → 削除（`disableSubmitButtons` 関数がボタン無効化を担当）

**自動生成ファイル**:

- `*_templ.go` ファイルは `.templ` ファイルから正しく生成されている
