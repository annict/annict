# ガイドライン同期コマンドの改善 設計書

## 実装ガイドラインの参照

この設計書はコマンドファイルの更新であり、Go版・Rails版の実装は伴いません。

## 概要

`/sync-guideline` コマンドのワークフローを改善する。

現状の `/sync-guideline` コマンドは、他プロジェクトのガイドラインを自プロジェクトに自動追記する方式だが、古い方針のガイドラインで新しいガイドラインを上書きしてしまうリスクがある。

新しい方式では、`docs/designs/template.md` をコピーした設計書の中に**差分レポート・採用方針・タスクリスト**をまとめ、ユーザーが方針を決定してからタスクリストを確定するフローに変更する。

**目的**:

- ガイドライン同期時に古い方針で新しい方針を上書きするリスクを排除する
- どの方針を採用するかをユーザーが明示的に判断できるようにする

**背景**:

- 現在の自動追記方式では、他プロジェクトの古いガイドラインが優先されてしまうケースがある
- プロジェクト間でガイドラインの方針が異なる箇所を可視化し、意思決定を明確にしたい

## 要件

### 機能要件

- `/sync-guideline` 実行時に `docs/designs/template.md` をコピーして設計書を作成する
- 設計書内に差分レポートセクション（Claude が記入）と採用方針セクション（ユーザーが記入）を設ける
- ユーザーが採用方針を追記した後、Claude がタスクリストを更新する
- `docs/templates/sync-guideline.md` は不要（設計書に統合）

### 想定されるワークフロー

1. ユーザーが `/sync-guideline` を実行
2. Claude が `docs/designs/template.md` をコピーして設計書を作成
   - 対象プロジェクト・比較対象ファイルの情報を記載
   - 各ガイドラインの差分を調査し、差分レポートセクションに記入
   - 採用方針セクションをユーザーが記入できる形で用意
   - タスクリストは空の状態で作成
3. ユーザーが差分を確認し、各項目に採用する方針を追記
4. Claude が方針に基づいてタスクリストを更新
   - プロジェクトごとにフェーズを設け、必要な修正をタスクとしてリストアップ

## 設計

### 設計書の構成（`/sync-guideline` が生成する設計書）

`docs/designs/template.md` をベースに、以下のセクションを追加・カスタマイズする。

#### 概要セクション

同期の目的と対象プロジェクトを記載する。

**対象プロジェクト**:

| プロジェクト | ディレクトリ |
|------------|------------|
| Annict     | `/workspace/` (自プロジェクト) |
| mewst      | `/mewst/` |
| wikino     | `/wikino/` |
| groobb     | `/groobb/` |

**比較対象ファイル**:

- CLAUDE.md ファイル: `CLAUDE.md`, `go/CLAUDE.md`, `rails/CLAUDE.md`
- Go版ガイドライン (`go/docs/`): `architecture-guide.md`, `handler-guide.md`, `i18n-guide.md`, `security-guide.md`, `templ-guide.md`, `validation-guide.md`

Claude が各ファイルの存在確認を行い、存在しないファイルにはマークを付ける。

#### 差分レポート・採用方針セクション

設計書の「設計」セクション内に配置する。Claude がファイルごとに差分を調査し、以下の形式で記入する。

```markdown
### 差分 1: [差分の概要]

**対象ファイル**: `go/docs/validation-guide.md`

**差分の内容**:
- Annict: [Annict での記述内容]
- mewst: [mewst での記述内容]
- wikino: [wikino での記述内容]

**採用方針**: <!-- ユーザーが記入 -->

---
```

差分がないファイルは「差分なし」と記載する。

#### タスクリストセクション

初回は空の状態で作成し、ユーザーが採用方針を記入した後に Claude が更新する。プロジェクトごとにフェーズを分け、各プロジェクトへの修正をタスクとしてリストアップする。

```markdown
### フェーズ 1: Annict (自プロジェクト)
- [ ] **1-1**: [修正内容]
  - ...

### フェーズ 2: mewst
- [ ] **2-1**: [修正内容]
  - ...

### フェーズ 3: wikino
- [ ] **3-1**: [修正内容]
  - ...
```

### `.claude/commands/sync-guideline.md` の構成

コマンドファイルに以下を記載する：

1. **対象プロジェクト一覧**: ディレクトリパスの定義
2. **比較対象ファイル一覧**: CLAUDE.md と docs/ 配下のファイル一覧
3. **手順**:
   - `docs/designs/template.md` をコピーして設計書を作成（ファイル名: `docs/designs/1_doing/sync-guideline-YYYYMMDD.md`）
   - 各ファイルの差分を調査し、差分レポートセクションに記入
   - 採用方針セクションをユーザーが記入できる形で用意
   - タスクリストは「ユーザーの採用方針決定後に更新」と記載
   - ユーザーに採用方針の記入を依頼するメッセージを出力
4. **注意事項**:
   - 存在しないファイルはスキップする
   - 差分がないファイルは「差分なし」と明記する
   - コンテキスト制限を回避するため、Task tool を使って並列で差分調査を行う

### 実装しない機能（スコープ外）

以下の機能は今回の実装では**実装しません**：

- **自動差分検出ツール**: 差分の検出は Claude が手動で行う（専用ツールは作らない）
- **groobb のガイドライン整備**: groobb にはまだ CLAUDE.md が存在しないため、同期対象外とする
- **Rails版 docs/ の同期**: Rails版の docs/ ディレクトリはどのプロジェクトにも存在しないため対象外
