## 概要

<!--
ガイドライン:

`animes` テーブル導入に伴い、新しい REST API V2 を提供する。V2 では作品とエピソードを統一的に `anime` リソースとして扱い、シンプルで一貫性のある API を提供する。

**目的**:

- `animes` テーブルを活用した新しい API エンドポイントを提供
- 作品とエピソードの統一的なアクセス方法を提供
- 将来の作品↔エピソード変換に対応した URL 設計

**背景**:

- 既存の REST API V1 は `works` と `episodes` を別々のリソースとして扱っている
- `animes` テーブル導入により、統一的な ID（`anime_id`）でアクセス可能になる
- 既存 API との互換性を維持しつつ、新しい API を段階的に提供する必要がある

### 設計方針

- **公開する ID は `animes.id` のみ**: `works.id` と `episodes.id` は内部的な ID として扱い、API では公開しない
- **内部的なフィールドは公開しない**: `parent_id` などは API レスポンスに含めない
- **関連リソースは展開して返す**: エピソードの親作品は `work` フィールド、前後のエピソードは `prev_episode` / `next_episode` フィールドで表現
- **単数形を採用**: 英語として自然な `/anime` を使用（`/animes` ではない）

### URL 設計

#### REST API V2 エンドポイント

```
GET /api/v2/anime/works             # 作品一覧
GET /api/v2/anime/:anime_id         # anime詳細（作品でもエピソードでも）
GET /api/v2/anime/:anime_id/episodes  # 特定作品のエピソード一覧
```

#### Web URL（参考）

```
GET /anime/:anime_id                # 作品詳細またはエピソード詳細
```

- 作品かエピソードかは `parent_id` で判別し、適切なテンプレートを表示
- **変換に強い**: 作品↔エピソード変換しても URL が変わらない

### API レスポンス設計

#### ID の表現ルール

- **トップレベルの `id`**: anime リソースを返すエンドポイントなので `anime_id` であることが自明。`id` のまま。
- **ネストされたリソースの ID**: `work_id` や `episode_id` と混同しないよう `anime_id` で明示する。

#### 作品の場合

```json
{
  "id": 123,
  "type": "work",
  "title": "作品タイトル",
  "title_kana": "サクヒンタイトル",
  "work": {
    "season_year": 2024,
    "media": "tv",
    "official_site_url": "https://example.com"
  }
}
```

#### エピソードの場合

```json
{
  "id": 456,
  "type": "episode",
  "title": "第2話 出会い",
  "episode": {
    "number": "2",
    "sort_number": 2
  },
  "work": {
    "anime_id": 123,
    "type": "work",
    "title": "作品タイトル"
  },
  "prev_episode": {
    "anime_id": 455,
    "title": "第1話 はじまり"
  },
  "next_episode": {
    "anime_id": 457,
    "title": "第3話 冒険"
  }
}
```

- トップレベルの `id` は anime リソースなので `anime_id` であることが自明
- ネストされたリソース（`work`, `prev_episode`, `next_episode`）は `anime_id` で明示し、`work_id` や `episode_id` と混同しないようにする
- 前後のエピソードが存在しない場合は `null` を返す

### 既存 URL からの移行

移行期間中は既存の URL から新しい URL へ 301 リダイレクトする：

```
/works/:work_id                     → /anime/:anime_id
/works/:work_id/episodes/:episode_id → /anime/:episode_anime_id
```

リダイレクトには `works.anime_id` と `episodes.anime_id` のマッピングを使用する。
